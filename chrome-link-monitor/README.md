# 链接变更监控

Chrome 扩展（Manifest V3）。在网页上对链接右键、地址栏输入 `lj`、或门户站点选即可加入监控列表，按设定间隔自动检查**正文**是否变化，并通过系统通知、弹窗、音效等方式提醒。

**当前版本：2.0.31**

**安装：** 在 `chrome://extensions` 加载本文件夹 `chrome-link-monitor`（须直接包含 `manifest.json`，不要选上一级目录）。

**配套 Android 应用：** 上级目录 [`link-monitor-android/`](../link-monitor-android/) — 手机端添加链接、定时检查、变更推送通知（独立运行，需手动配置相同 URL）。打包见该目录 README。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| **多链接并行监控** | `monitors[]` 列表，每条独立基准、间隔、启停；列表显示备注名或默认编号 |
| **一键添加** | 面板输入 URL → **添加并开始监控**，自动建基准并启动闹钟 |
| **检查间隔** | 全局统一，或为每条链接单独设置；支持秒～周 |
| **多种添加入口** | 面板、右键菜单、地址栏 `lj`、门户点选工具条 |
| **正文 vs 噪声** | 正文 core 变 → 强提醒；仅广告/浮层变 → 轻量通知（可忽略） |
| **门户站深度适配** | `ahtba.org.cn`、`ggzy.hefei.gov.cn`：点选模式、DOM 指纹、标签页优先读取 |
| **登录态页面** | 门户从已打开标签页读内容；普通站 `fetch`（带 Cookie）+ 标签页回退 |
| **强提醒分层** | 系统通知 + Chrome 弹窗 + 网页遮罩 + 三声提示音 + 扩展角标 |
| **弹窗节流** | 60 分钟内强提醒弹窗超 40 次可询问暂停；测试通知不计入 |
| **启停生命周期队列** | 快速开始/停止不竞态；停止优先，可打断进行中的检查 |
| **可选门户 UI** | 右下角点选工具条、页面「可监控」标记均可开关 |
| **内置测试页** | uuid 链接验证变更检测；example.com 验证无变化静默 |

## 安装

1. 打开 Chrome → `chrome://extensions/`
2. 开启 **开发者模式**
3. **加载已解压的扩展程序** → 选择 **`chrome-link-monitor`** 文件夹
4. Windows：**设置 → 系统 → 通知** → 允许 Chrome 通知

> 若加载的是上一级目录（看不到 `manifest.json`），扩展将无法工作。

### Windows 路径注意

扩展路径含中文等非 ASCII 字符时，Chrome 可能无法正确加载 Service Worker（表现为 `importScripts` / `messages.js` 报错，或后台「无效」且功能全失效）。

**建议：** 将整个文件夹复制到纯英文路径，例如 `D:\chrome-link-monitor`，再从该路径加载。

安装前可运行：

```bash
python verify_extension.py
```

## 使用方法

### 扩展面板（工具栏图标）

#### 添加监控链接

- **备注名称**（可选）
- **链接 URL**
- **添加并开始监控**：立即加入「正在监控」、建立指纹基准、启动定时检查；输入框清空后可继续添加

新链接使用当前 **全局检查间隔**（见「正在监控」区域）。

#### 正在监控

- 状态行：正在监控的链接数量
- **所有链接使用相同检查间隔**
  - **勾选**：显示全局间隔（预设 5 秒/5 分/30 分等、保存检测间隔）
  - **不勾选**：每条链接第二行显示独立间隔与「保存间隔」
- 每条监控（两行布局）：
  - **第一行**：链接框（仅显示 **备注名** 或 **N号链接**；不展示 URL，悬停可看完整地址）+ **停止**
  - **第二行**（仅非全局间隔时）：该链接的检测间隔设置

点击链接框打开对应页面。

#### 弹窗提醒

| 选项 | 说明 |
|------|------|
| 60 分钟内弹窗超过 40 次时询问 | 防止强提醒刷屏 |
| 暂停 Chrome 弹窗与网页遮罩 | 仍保留系统通知 |
| 忽略纯广告/弹窗变化 | 仅正文变更时强提醒 |

#### 启动行为

| 选项 | 说明 |
|------|------|
| 显示右下角点选工具条 | 门户站专用；**默认关闭**，需勾选 |
| 显示「可监控」标记 | 门户页链接旁红字提示（不可点，请用点选） |
| 打开浏览器时自动打开监控页面 | Chrome 启动时打开已监控 URL |
| 选择链接后自动打开监控页面 | 注册/点选监控时后台打开对应标签页（与上一项独立） |

#### 底部操作

- **打开测试页** · **立即检查** · **测试通知**（不计入 40 次限额）· **停止全部监控**

### 任意网站

| 方式 | 操作 |
|------|------|
| **右键链接** | 开启链接监控 |
| **右键页面 / 扩展图标** | 监控当前标签页 |
| **地址栏 `lj`** | 输入 URL 回车添加；空回车监控当前标签 |

### 安徽省招标网 / 合肥公共资源（门户站）

适配：`www.ahtba.org.cn`、`ahtba.org.cn`、`ggzy.hefei.gov.cn` 及子域。

这些站点常 **禁用或限制链接右键**，请用点选：

1. 面板「启动行为」勾选 **显示右下角点选工具条**
2. 打开列表页，工具条点 **开启点选模式**
3. 点击要监控的 **列表条目**（点选拦截跳转并注册监控，默认间隔，约 **3 秒** 后自动退出点选）
4. 工具条 **×** 可关闭；右上角 **关闭点选模式** 可手动退出

**建议监控详情页 URL**，勿只监控首页列表（轮播、统计数字、其他条目更新易误报）。

## 工作原理（详细）

### 整体架构

| 组件 | 文件 | 职责 |
|------|------|------|
| **Service Worker** | `background/background.js` | 闹钟、抓取/读页、指纹比对、通知、右键菜单、`lj` |
| **设置面板** | `popup/` | 添加链接、间隔、启停、各项开关 |
| **门户 content script** | `content/site-ahtba.js` | 可选工具条、点选模式、「可监控」标记 |
| **指纹注入回退** | `content/portal-extract.js` | 向标签页注入提取逻辑 |
| **提醒页** | `alert/`、`offscreen/` | Chrome 弹窗、节流询问、提示音 |

普通网站 **不需要** content script，后台 `fetch` 或读已打开标签页即可。

Service Worker 由 `bundle_sw.py` 打包为 **单文件**（模块：`messages.js`、`page-fingerprint.js`、`monitors.js` + `background.entry.js`），**不使用** `importScripts`，避免 Windows 路径编码问题。业务逻辑改 `background.entry.js` 后执行 `python bundle_sw.py`。

### 一次检查的完整流程

```
chrome.alarms 到时 / 用户点「立即检查」
        ↓
读取 chrome.storage.local：monitors[]、间隔、上次指纹基准
        ↓
对每条「到期」监控执行 checkSingleMonitor
        ↓
┌─ 门户站（ahtba / ggzy）────────────────────────────┐
│ 1. 查找 URL 与监控地址一致的标签页                  │
│ 2. 可选：勾选「选择链接后自动打开」则创建后台标签   │
│ 3. 向标签页发消息或注入 portal-extract.js 取 DOM 指纹 │
│ 4. 无标签且 fetch 失败 → 提示保持标签页打开         │
└──────────────────────────────────────────────────────┘
        ↓（非门户站）
┌─ 普通站 ────────────────────────────────────────────┐
│ 1. 后台 fetch URL（credentials: include, no-store）│
│ 2. 失败则尝试已打开且 URL 匹配的标签页              │
│ 3. 503/504/Failed to fetch 自动重试               │
└──────────────────────────────────────────────────────┘
        ↓
对 raw / core / overlay 分别计算哈希（djb2 变体 → base36）
        ↓
与 lastCoreContent / lastRawContent / lastOverlaySignature 比对
        ↓
首次检查 → 只建基准，不提醒
正文 core 变 → 强提醒 + 更新基准
仅 raw/overlay 变 → 轻量通知或忽略（看设置）
完全相同 → 静默，只更新 lastCheckAt
```

### 三种指纹

| 指纹 | 存储字段 | 如何提取 | 用途 |
|------|----------|----------|------|
| **正文 core** | `lastCoreContent` | 去 script/style 后的主内容；门户站拼接 `LIST::` 列表行快照 | **决定是否强提醒** |
| **整页 raw** | `lastRawContent` | 完整 HTML / outerHTML 哈希 | 判断「只有装饰变了」 |
| **浮层 overlay** | `lastOverlaySignature` | 高 z-index 的 fixed/absolute 大块元素摘要 | 识别弹窗广告层 |

门户站（高敏感主机）额外处理：

- 优先从 `table tbody`、`affiche`、`list-box`、`el-table` 等选择器取列表区
- `stabilizeVolatilePortalText` 过滤「今日发布/抽取」统计、「数据加载中」、扩展工具条文字
- **监控 URL 须与打开的标签页 URL 一致**，避免「开着首页却监控详情」导致误报

### 多链接与启停

- 每条监控：`id`、`number`、`url`、`name`、`intervalMinutes`、三类指纹基准、`lastCheckAt`
- `useGlobalInterval` 为 true 时，闹钟按全局最小间隔调度；为 false 时每条按各自 `intervalMinutes` 到期
- **开始/停止** 走生命周期队列：`lifecycleStopRequested` 可打断批量检查
- 生命周期 toast（已添加/已停止）约 **1 秒内最多 2 条**，防刷屏
- **停止优先**：用户停止后，进行中的检查尽快中止，不再发「已变更」类提醒

### 提醒分层

| 级别 | 触发 | 渠道 |
|------|------|------|
| **强提醒** | core 变化 | 系统通知 + Chrome 弹窗 + 网页遮罩 + offscreen 音效 + 角标「变」 |
| **轻量提醒** | 仅 raw/overlay 变化 | 系统通知（标题注明可能为广告/弹窗） |
| **失败提醒** | 非瞬时错误且检查失败 | 系统通知（同一 URL **15 分钟**内失败通知降噪） |
| **静默** | 无变化 / 忽略噪声 / 门户瞬时跳过 | 无用户可见提醒 |

系统通知图标使用 `icons/icon-notification256.png`（256×256），通过 `chrome.runtime.getURL` 引用，避免在 SW 内嵌大段 base64 导致加载失败或模糊。

Chrome 弹窗 + 网页遮罩在 60 分钟内累计 **40 次** 会弹出 `ask-throttle.html` 询问是否暂停；**测试通知** 不计入。

### 数据存储

| 数据 | 位置 | 说明 |
|------|------|------|
| 监控列表、间隔、指纹基准、各开关 | `chrome.storage.local` | 持久 |
| 点选会话（门户「已选项」） | `linkMonitorPickedSession` | 后台在注册时同步 |
| 工具条本页关闭状态 | 页面 `sessionStorage` | `lm-ahtba-bar-closed` |
| 门户页面临时 UI 状态 | content script 内存 | 不依赖 `chrome.storage.session` |

## 正文 vs 广告/弹窗

| 检测结果 | 提醒 | 「立即检查」面板反馈 |
|----------|------|---------------------|
| 正文 core 变化 | 强提醒 | 检测到正文内容变化 |
| 仅 raw/overlay 变化 | 轻量系统通知 | 可能仅为广告/弹窗变化 |
| 勾选「忽略纯广告/弹窗」 | 静默更新基准 | 广告/弹窗变化已忽略 |
| 无变化 | 静默 | 正文无变化 |
| 首次建立基准 | 静默 | 正文无变化 |

**没变化却收到通知？** 常见原因：广告/弹窗轮换、监控列表页而非详情、门户两次抓取来源不一致（HTTP vs 标签页）、测试 uuid 本身每次变。对策：勾选忽略噪声、加大间隔、改监控详情 URL、门户保持对应标签页打开。

调试：Service Worker 控制台搜索 `正文内容变更`、`仅装饰/广告变化`、`内容无变化`、`检查失败`。

## 适用场景与建议

### 政府招投标 / 公告详情

**适合：** 单个项目 **详情页**（中标、答疑、成交结果等）；页面上有可见状态文字。

**不建议：** 只监控首页列表；监控 PDF 附件内文字；门户站间隔过短（日常建议 ≥ 30 分钟，测试可用 5 秒 + uuid）。

**推荐流程（ahtba）：**

1. 打开 [首页](https://www.ahtba.org.cn/site/home/index) 或列表
2. 勾选显示点选工具条 → 点选模式 → 点 **某条公告**（不是普通点击跳转）
3. 保持浏览器登录；可选勾选「选择链接后自动打开监控页面」

### 新闻 / 商品 / 普通网页

1. 打开 **详情页**
2. 右键或 `lj` 添加监控
3. 需登录站点：**保持该标签页打开**

### 验证扩展是否正常

内置测试页 + [httpbingo.org/uuid](https://httpbingo.org/uuid)：间隔 5 秒时第二次检查应强提醒；[example.com](https://example.com) 应静默。

## 重载扩展后

1. `chrome://extensions` → **重新加载**
2. **错误 → 全部清除**（清除旧版残留报错）
3. 所有监控相关页面 **F5 / Ctrl+Shift+R**
4. 工具条标题应含 `v2.0.29`（来自 manifest）
5. 指纹逻辑升级后，误报条目建议 **停止后重新添加** 重建基准

`Extension context invalidated` = 页面仍在跑旧 content script，必须刷新。

Service Worker 显示 **（无效）** 多为休眠，非故障；点「检查视图」可看日志，首行应为 `// BUILD: 2.0.29`。

## 常见问题与解决办法

### 安装与加载

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 扩展无法加载 | 文件夹选错 | 必须选含 `manifest.json` 的 `chrome-link-monitor` |
| `importScripts` / messages.js 报错 | 旧版或多实例；中文路径 | 移除旧扩展；复制到 `D:\chrome-link-monitor`；用 2.0.17+ |
| 门户无工具条 | 默认不显示 | 「启动行为」勾选显示工具条；F5 |
| 重载后点选无效 | 旧 content script | Ctrl+Shift+R |

### 点选模式

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 点选未加入监控 | 通信失败 | 重载扩展 + 刷新 |
| 未识别链接 | 区域无 href/onclick | 点条目正文或链接文字 |
| 点选后关不掉弹窗 | 旧版拦截过宽 | 用「关闭点选模式」或升级 2.0.4+ |

### 检查与误报

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 没变化也报变更 | 列表页/噪声/基准错误 | 详情 URL；忽略噪声；重新添加 |
| 第一次变、第二次不变 | 首次对齐基准 | 正常，以第二次为准 |
| 变了不提醒 | PDF/纯 JS/未进 HTML | 保持详情标签页 |
| 门户误报多 | 动态列表、浮层 | 忽略噪声；加大间隔 |
| 检查失败 | 未登录、限流 | 保持标签页；门户勿仅靠 fetch |

### 通知与弹窗

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 无系统通知 | Windows 禁通知 | 系统设置允许 Chrome |
| 图标模糊 | 曾用 16px 图标 | 2.0.20+ 已用 256px PNG |
| 弹窗过多 | uuid 测试或变更频繁 | 暂停弹窗；加大间隔 |

### 多链接

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 只监控了一条 | 未添加成功 | 看「正在监控」列表；点选/右键会自动添加 |
| 停止后仍检查片刻 | 队列延迟 | 稍等；停止优先 |
| 换监控目标 | 旧条目仍在 | 单独停止或全部停止后重加 |

## 能力边界（做不到的事）

- **不解析语义**：不知道「状态=成交」，只比指纹
- **不读附件**：PDF、Word 等附件内文字变更检测不到
- **依赖 HTML**：Canvas、WebSocket 且不落 DOM 的变更可能漏报
- **闹钟精度**：后台休眠时秒级间隔可能漂移
- **不绕过反爬**：站点封禁时只能依赖已打开标签页
- **网页遮罩**：仅 http(s) 页面；`chrome://` 无法遮罩

请遵守目标网站服务条款与 robots 规则，合理设置间隔，避免对站点造成压力。
