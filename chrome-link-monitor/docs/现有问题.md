## 一、安装与加载

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 扩展完全无法工作 | 加载了上一级目录（无 `manifest.json`） | 必须选择含 `manifest.json` 的 **`chrome-link-monitor`** 文件夹 |
| `importScripts` / `messages.js` 报错 | 旧版扩展；或路径含中文等非 ASCII | 移除旧实例；复制到 `D:\chrome-link-monitor`；使用 2.0.17+ 打包版 SW |
| Service Worker 显示「无效」 | 多为休眠，非必然故障 | 点「检查视图」看日志；有操作时会唤醒；首行应为 `// BUILD: 2.0.29` |
| `verify_extension.py` 警告路径非 ASCII | 中文路径可能导致 SW/资源加载失败 | 复制到英文路径后重新加载；运行 `python verify_extension.py` |
| 改过 `background.entry.js` 后行为异常 | 未重新打包 SW | 执行 `python bundle_sw.py` 后重载扩展 |

## 二、重载扩展 / 升级版本后

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| `Extension context invalidated` | 页面仍运行旧 content script | 所有监控相关页面 **F5 或 Ctrl+Shift+R** |
| 点选无反应、工具条版本不对 | 未刷新门户页 | 重载扩展 + 强刷页面；工具条应显示当前 manifest 版本 |
| 旧报错残留 | 扩展卡片错误列表未清 | `chrome://extensions` → 错误 → **全部清除** |
| 升级后误报增多 | 指纹逻辑变更，旧基准不适用 | **停止误报条目后重新添加**，重建基准 |
| 说明书界面与实装不一致 | `说明书.md` 部分章节版本滞后 | 以面板实际界面与 `README.md` 为准 |

## 三、通知与弹窗

提醒分三层：**Windows 系统通知**、**Chrome 扩展弹窗**（`alert.html`）、**网页遮罩**。强提醒理论上三层都有；轻量提醒（仅广告/浮层）**只有系统通知**。

### 3.1 常见现象

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 完全没有系统通知 | Windows 禁用了 Chrome 通知 | 设置 → 系统 → 通知 → 允许 **Google Chrome** 并开启 **横幅** |
| 只有 Chrome 弹窗，没有 Windows 通知 | 系统/Chrome 通知被拦；或通知只在操作中心 | 同上；检查专注助手/请勿打扰；见 [故障排查](./故障排查-弹窗不弹.md) 第四节 |
| 只有 Windows 通知，没有 Chrome 弹窗 | 勾选了「暂停 Chrome 弹窗与网页遮罩」 | 扩展面板 → 弹窗提醒 → **取消勾选** |
| 弹窗过多 | uuid 测试、变更频繁、间隔过短 | 加大间隔；勾选忽略噪声；或暂停 Chrome 弹窗（仍保留系统通知） |
| 60 分钟后突然只剩系统通知 | 弹窗超 40 次后点了「暂停弹窗提醒」 | 取消「暂停 Chrome 弹窗」选项 |
| 通知图标模糊 | 旧版使用过小图标 | 2.0.20+ 已用 256px PNG；确认版本并重载 |
| 「链接监控通知创建失败」 | `chrome.notifications` 失败（路径/权限） | 英文路径；检查通知权限；看 SW 控制台 |
| 「Chrome popup alert failed」 | `chrome.windows.create` 失败 | 重载扩展；确认 `windows` 权限；特殊环境（kiosk/远程桌面）可能限制新窗口 |
| 网页遮罩不出现 | 无普通 http(s) 标签；或已暂停弹窗 | 打开普通网页标签；取消暂停弹窗；PDF/chrome:// 页无法遮罩 |
| 测试通知与真实提醒表现不同 | 测试通知不计入 40 次节流 | 用「测试通知」验通道；真实变更受节流与级别影响 |

### 3.2 设计上的「不弹」（非故障）

| 情况 | 说明 |
|------|------|
| 首次建立基准 | 只存指纹，不提醒 |
| 内容无变化 | 静默 |
| 仅广告/浮层变化 + 勾选「忽略噪声」 | 静默更新基准 |
| 轻量提醒（noise） | **不弹** Chrome 弹窗与遮罩，仅系统通知 |
| 检查间隔未到 | 定时检查跳过；「立即检查」可强制 |
| 失败通知 15 分钟冷却 | 同一 URL 重复失败不重复弹 |
| 生命周期通知 1 秒内超 2 条 | 「已添加/已停止」被节流 |

完整列表见 [故障排查-弹窗不弹.md](./故障排查-弹窗不弹.md)（47 条）。

## 四、检查与误报

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 没变化也报变更 | 监控列表页/首页；广告轮播；基准错误 | 改监控 **详情页 URL**；勾选忽略噪声；停止后重新添加 |
| 第一次变、第二次不变 | 首次检查对齐基准 | 正常，以第二次为准 |
| 正文变了却不提醒 | 变更在 PDF/纯 JS；未进 HTML DOM | 监控 HTML 详情页；保持标签页打开 |
| 门户站误报多 | 动态列表、浮层、统计数字 | 忽略噪声；间隔 ≥ 30 分钟（日常）；勿只监控首页 |
| 开着首页却监控详情 URL | 门户抓取来源与标签页不一致 | **监控 URL 须与打开的标签页 URL 一致** |
| httpbingo uuid 一直提醒 | 每次内容本就不同 | 正常；测试完可停止或换 example.com |
| example.com 不应提醒却提醒 | 基准未建立或噪声 | 连续「立即检查」两次对比；查 SW 日志 |

**没变化却收到通知的常见原因：** 广告/弹窗轮换、监控列表页、门户 HTTP vs 标签页抓取不一致、uuid 测试链本身每次变。

**调试：** Service Worker 控制台搜索 `正文内容变更`、`仅装饰/广告变化`、`内容无变化`、`检查失败`。

## 五、检查失败

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 检查失败 / 503 / 504 / 502 | 站点限流、网关超时 | 加大间隔；稍后重试；保持标签页打开 |
| `Failed to fetch` | 网络瞬断、CORS、站点拒绝 | 自动重试；门户保持标签页 |
| 门户「页面加载中，跳过本次」 | 高敏感站 core 未就绪 | 保持对应标签页打开并登录；加大间隔 |
| 需要登录的站失败 | 后台 fetch 无会话 | 浏览器登录后 **保持该页标签打开** |
| httpbin 失败 | 服务已废弃不稳定 | 改用 **https://httpbingo.org/uuid** |
| 同一 URL 短时间不再报失败 | 15 分钟失败通知降噪 | 正常；等冷却或修网络/登录 |
| `(skipped, will retry)` 日志 | 瞬时错误被跳过 | 等下次检查，非永久故障 |

## 六、门户站（ahtba / ggzy）

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 没有右下角工具条 | **默认不显示** | 面板「启动行为」勾选 **显示右下角点选工具条**；F5 |
| 点选后没加入监控 | content script 通信失败 | 重载扩展 + Ctrl+Shift+R |
| 点选未识别链接 | 区域无 href/onclick | 点条目正文或链接文字区域 |
| 点选后弹窗关不掉 | 旧版拦截过宽 | 用「关闭点选模式」；升级 2.0.4+ |
| 链接右键无效 | 站点禁用或限制右键 | 用 **点选模式** 或页面空白处右键 |
| 工具条点选后约 3 秒自动退出 | 设计行为 | 需继续点选可再次开启点选模式 |
| 监控失败 toast | 注册通信失败 | 刷新页面后重试 |

**不建议：** 只监控首页列表；监控 PDF；门户间隔过短（日常建议 ≥ 30 分钟）。

## 七、添加监控入口

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 地址栏不能右键监控 | **Chrome 限制**，地址栏无法挂扩展右键菜单 | 用地址栏 **`lj`**、页面右键或面板输入 URL |
| 普通站没有工具条 | 设计如此，仅门户站有 | 右键 / `lj` / 面板添加 |
| `lj` 无效 URL | 输入格式错误 | 输入完整 http(s) URL；空回车监控当前标签 |
| 添加后列表未更新 | 添加失败或未开始监控 | 看面板 toast；确认「正在监控」数量 |

## 八、多链接与启停

| 现象 | 原因 | 解决办法 |
|------|------|----------|
| 只监控了一条 | 未添加成功 | 查看「正在监控」列表；点选/右键应自动添加并开始 |
| 停止后仍检查片刻 | 生命周期队列延迟 | 稍等；**停止优先**会打断进行中的检查 |
| 换监控目标旧条目仍在 | 未单独停止 | 点该条 **停止** 或 **停止全部监控** 后重加 |
| 快速启停通知刷屏 | 生命周期通知节流 | 1 秒内最多 2 条，属正常防刷屏 |

## 九、平台与环境限制

| 限制 | 说明 |
|------|------|
| **中文等非 ASCII 安装路径** | Chrome 可能无法正确加载 SW 与通知图标 |
| **Windows 通知策略** | 企业组策略、安全软件可只拦系统通知，不拦扩展弹窗 |
| **专注助手 / 请勿打扰** | 通知可能只进操作中心，不弹横幅 |
| **Chrome 后台休眠** | 秒级间隔可能漂移；长时间无操作需「测试通知」唤醒 SW |
| **多显示器** | 扩展弹窗可能出现在副屏 |
| **全屏应用** | Chrome 弹窗可能被盖住 |
| **kiosk / 部分远程桌面** | 可能限制 `windows.create` 新窗口 |
| **不同 Chrome 配置文件** | `chrome.storage.local` 不共享，设置与监控列表各自独立 |

## 十、能力边界（做不到的事）

| 边界 | 说明 |
|------|------|
| **不解析语义** | 不知道「状态=成交」，只比页面指纹哈希 |
| **不读附件** | PDF、Word 等附件内文字变更检测不到 |
| **依赖 HTML/DOM** | Canvas、纯 WebSocket 且不落 DOM 的变更可能漏报 |
| **闹钟精度** | 后台休眠时极短间隔不准确 |
| **不绕过反爬** | 站点封禁后台 fetch 时只能依赖已打开标签页 |
| **网页遮罩范围** | 仅 http(s) 页面；`chrome://`、`chrome-extension://` 无法遮罩 |
| **门户域名有限** | 深度适配仅 `ahtba.org.cn`、`ggzy.hefei.gov.cn` 及子域 |
| **不替代人工合规判断** | 请遵守目标网站服务条款与 robots，合理设置间隔 |

## 十二、快速自检清单

- [ ] 加载目录正确（含 `manifest.json`）
- [ ] 路径尽量为英文（如 `D:\chrome-link-monitor`）
- [ ] `python verify_extension.py` 无 ERROR
- [ ] Windows 与 Chrome 均允许通知
- [ ] 面板「正在监控」≥ 1
- [ ] 未误勾「暂停 Chrome 弹窗」
- [ ] 点「测试通知」：系统通知 + Chrome 弹窗 + 音效
- [ ] 门户页已勾选工具条并强刷页面
- [ ] 监控的是 **详情页** 而非首页列表
