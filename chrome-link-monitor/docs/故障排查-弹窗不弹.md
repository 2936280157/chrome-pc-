# 故障排查：不弹弹窗的所有原因与解决办法

本文档列举「链接变更监控」扩展中**所有可能导致提醒不出现**的原因及对应解决办法。

## 提醒渠道说明

本扩展的「弹窗」分三层，很多情况是**只缺其中一层**，要先分清缺的是哪一种：

| 渠道 | 表现 |
|------|------|
| **Windows 系统通知** | 右下角任务栏旁 / 操作中心的 Chrome 通知 |
| **Chrome 扩展弹窗** | 独立小窗口（`alert/alert.html`，标题「链接变更提醒」） |
| **网页遮罩** | 当前网页上的红色遮罩层 |

强提醒（正文变化）理论上三层都有；轻量提醒（仅广告/浮层）**只有系统通知，没有 Chrome 弹窗和遮罩**——这是设计如此。

## 一、本来就不会弹（正常行为，不是故障）

| # | 原因 | 说明 | 解决办法 |
|---|------|------|----------|
| 1 | **页面内容没有变化** | core / raw / overlay 指纹与基准一致 | 正常；用测试页 uuid 链接验证 |
| 2 | **首次建立基准** | `lastCoreContent === null` 时只存基准，不提醒 | 等第二次检查；或点「立即检查」两次 |
| 3 | **仅广告/浮层变化（轻量提醒）** | `level: 'noise'`，代码直接 `return`，**不弹 Chrome 弹窗/遮罩** | 若需要强提醒，监控**详情页**而非列表页；或取消「忽略噪声」看系统通知 |
| 4 | **勾选了「忽略纯广告/弹窗变化」** | 仅 raw/overlay 变时静默更新基准 | 扩展面板 → 弹窗提醒 → **取消勾选** |
| 5 | **检查间隔未到** | 定时闹钟触发时，该链接尚未到期（`isMonitorDue`） | 点「**立即检查**」可跳过间隔；或缩短间隔 |
| 6 | **门户站页面还在加载** | 高敏感站点抓不到 core 时跳过本次（日志：`页面加载中，跳过本次`） | 保持对应标签页打开并登录；加大间隔后重试 |
| 7 | **瞬时网络错误被跳过** | 503/504/502/429、`Failed to fetch`、门户无标签页等只打日志 `(skipped, will retry)` | 等下次检查；修复网络/登录/标签页 |
| 8 | **检查失败但在 15 分钟冷却内** | 同一 URL 失败通知 15 分钟内只发一次 | 等冷却结束；看 Service Worker 日志 `(notification suppressed)` |
| 9 | **监控已停止 / 列表为空** | `monitoringActive === false` 或无 monitors | 重新「添加并开始监控」 |
| 10 | **检查过程中被停止打断** | `lifecycleStopRequested` 中止，不发变更提醒 | 停止后不要再期待当次结果；重新启动监控 |
| 11 | **生命周期通知被节流** | 1 秒内最多 2 条「已添加/已停止」类通知 | 正常防刷屏；稍后再试 |

## 二、扩展设置导致 Chrome 弹窗/遮罩不弹（系统通知仍可能有）

| # | 原因 | 说明 | 解决办法 |
|---|------|------|----------|
| 12 | **勾选了「暂停 Chrome 弹窗与网页遮罩」** | `popupsSuppressed === true` 时，`broadcastAlert` 在系统通知后直接 return | 扩展面板 → 弹窗提醒 → **取消勾选** |
| 13 | **60 分钟内弹窗超 40 次后点了「暂停弹窗提醒」** | 节流询问里选 pause 会写入 `popupsSuppressed: true` | 同上，取消勾选；或等 60 分钟后重新评估 |
| 14 | **手动在 storage 里残留暂停状态** | 换电脑/重装后若同步了旧设置 | 打开面板确认上述两个选项；必要时在 `chrome://extensions` 清除扩展数据后重配 |

> **注意：** 以上只关 **Chrome 弹窗 + 网页遮罩**，**不关** Windows 系统通知。

## 三、扩展逻辑：某次强提醒被「换成别的」或「少一层」

| # | 原因 | 说明 | 解决办法 |
|---|------|------|----------|
| 15 | **弹窗节流询问替换了当次 alert** | 60 分钟内强提醒弹窗 >40 次时，弹出 `ask-throttle.html` 而不是 `alert.html` | 若选了「继续」则后续仍有弹窗；若选了「暂停」见 #13 |
| 16 | **节流询问在冷却期内** | `throttleAskCooldownUntil` 未过期时不再弹询问 | 正常；冷却 60 分钟 |
| 17 | **测试通知 vs 普通提醒** | 「测试通知」走 `skipThrottle: true`，不受 40 次限制 | 用「测试通知」验证通道是否正常 |
| 18 | **失败提醒级别为 `fail`** | 有系统通知 + 音效 + 角标「!」，**仍有** Chrome 弹窗（除非 #12） | 若完全无反应，查 #28 扩展是否存活 |

## 四、Windows 系统通知不弹

| # | 原因 | 解决办法 |
|---|------|----------|
| 19 | **Windows 总通知关闭** | 设置 → 系统 → 通知 → 打开「通知」 |
| 20 | **Chrome 通知权限关闭** | 同上路径 → 找到 **Google Chrome** → 打开，并勾选 **显示通知横幅** |
| 21 | **专注助手 / 请勿打扰开启** | Win11 快速设置关闭；或在通知设置里允许 Chrome 突破专注助手 |
| 22 | **Chrome 站内通知被禁** | `chrome://settings/content/notifications` → 允许 |
| 23 | **扩展无「通知」权限** | `chrome://extensions` → 本扩展 → 确认权限含 notifications |
| 24 | **`chrome.notifications.create` 失败** | Service Worker 控制台搜 **「链接监控通知创建失败」**；扩展会重试无图标版，再失败走 offscreen 兜底 |
| 25 | **扩展路径含中文等非 ASCII** | 复制到 `D:\chrome-link-monitor` 等英文路径后重新加载；运行 `python verify_extension.py` |
| 26 | **企业策略 / 安全软件拦截** | 联系 IT 放行 Chrome 通知；或暂时依赖 Chrome 扩展弹窗（#12 勿勾选） |
| 27 | **通知在操作中心但不弹横幅** | 打开任务栏右侧 **通知中心** 查看；调整 Chrome 为「横幅 + 声音」 |
| 28 | **Service Worker 未运行 / 扩展无效** | `chrome://extensions` → 本扩展 → Service Worker 是否报错/灰色；重载扩展 |

## 五、Chrome 扩展弹窗（`alert.html` 小窗口）不弹

| # | 原因 | 解决办法 |
|---|------|----------|
| 29 | **见 #12～#13** | 设置里取消「暂停 Chrome 弹窗」 |
| 30 | **见 #3 轻量提醒** | 只有正文 core 变化才弹此窗口 |
| 31 | **`chrome.windows.create` 失败** | Service Worker 控制台搜 **`Chrome popup alert failed`**；重载扩展；确认有 `windows` 权限 |
| 32 | **Chrome 未在前台 / 被其他全屏应用盖住** | 弹窗 `focused: true` 但仍可能被全屏游戏等挡住；Alt+Tab 找 Chrome 小窗 |
| 33 | **多显示器弹在另一块屏** | 检查副屏任务栏 |
| 34 | **Chrome 以特殊模式运行** | 某些 kiosk/远程桌面环境限制新窗口；换本地正常 Chrome 测试 |
| 35 | **扩展后台被系统休眠杀死** | 长时间无操作后先点「测试通知」唤醒 SW；Windows 电源选项勿过度休眠浏览器 |

## 六、网页遮罩不弹（但 Chrome 小窗可能有）

| # | 原因 | 解决办法 |
|---|------|----------|
| 36 | **见 #12～#13** | 取消暂停弹窗 |
| 37 | **当前只有 `chrome://` / 扩展页标签** | 遮罩只注入普通 http(s) 页；无可用标签则跳过（代码静默 `.catch`） |
| 38 | **当前标签是 PDF、Chrome 网上应用店等** | 脚本注入失败被静默忽略；看 Chrome 弹窗即可 |
| 39 | **页面 CSP 极严** | 少数站点禁止注入；同样依赖 Chrome 弹窗 |
| 40 | **没有打开任何普通网页** | 至少保留一个 http(s) 标签；或接受只有系统通知 + 扩展小窗 |
| 41 | **注入失败无报错** | 属预期（`.catch(function () {})`）；用「测试通知」在有普通网页打开时试 |

## 七、扩展整体失效 → 什么都不弹

| # | 原因 | 解决办法 |
|---|------|----------|
| 42 | **加载目录错误** | 必须选含 `manifest.json` 的 **`chrome-link-monitor`** 文件夹，不是上一级 |
| 43 | **Service Worker 加载失败（中文路径）** | 移到英文路径；看 SW 控制台是否有 `importScripts` / 资源 404 |
| 44 | **未重新 bundle** | 若改过 `background.entry.js`，需 `python bundle_sw.py` 后重载 |
| 45 | **Chrome 版本过旧** | MV3 + offscreen 需较新 Chrome；升级后重试 |
| 46 | **监控从未真正启动** | 面板「正在监控」数量应为 ≥1；添加后应自动开始 |
| 47 | **看错机器/配置文件** | 扩展设置存在 `chrome.storage.local`，不同 Chrome 配置互不影响 |

## 八、快速排查流程

```
1. 扩展面板 → 点「测试通知」
   ├─ 全无 → #42～#47（扩展/SW/路径）
   ├─ 只有系统通知 → #29～#35（Chrome 弹窗）+ 查 #12
   ├─ 只有 Chrome 弹窗 → #19～#28（Windows 通知）
   └─ 三者都有 → 扩展通道正常，问题在「检测未触发提醒」→ #1～#11

2. chrome://extensions → Service Worker → 控制台
   ├─ 「链接监控通知创建失败」→ #24～#25
   ├─ 「Chrome popup alert failed」→ #31
   ├─ 「正文内容变更」但无弹窗 → #12 或 #29
   ├─ 「仅装饰/广告变化」→ #3 或 #4
   ├─ 「内容无变化」→ #1
   └─ 「(skipped, will retry)」→ #7

3. 扩展面板检查
   ├─ 「暂停 Chrome 弹窗与网页遮罩」→ 取消
   ├─ 「忽略纯广告/弹窗变化」→ 按需取消
   └─ 正在监控数量 > 0
```

## 九、对照表：什么情况会有什么提醒

| 场景 | 系统通知 | Chrome 弹窗 | 网页遮罩 | 音效 |
|------|:--------:|:-----------:|:--------:|:----:|
| 正文 core 变化（强提醒） | ✅ | ✅* | ✅* | ✅ |
| 仅广告/浮层变化 | ✅ | ❌ | ❌ | ❌ |
| 忽略噪声 + 仅广告变化 | ❌ | ❌ | ❌ | ❌ |
| 检查失败 | ✅ | ✅* | ✅* | ✅ |
| 无变化 / 首次基准 / 跳过 | ❌ | ❌ | ❌ | ❌ |
| 已暂停 Chrome 弹窗 | ✅ | ❌ | ❌ | 看级别 |

\* 除非勾选「暂停 Chrome 弹窗与网页遮罩」。
